## Web SDK (React)

### Install

`@revturbine/sdk` is published to the public npm registry — no auth, token, or registry configuration required.

<Tabs>
  <TabItem label="npm">
  ```bash
  npm install @revturbine/sdk
  ```
  </TabItem>
  <TabItem label="pnpm">
  ```bash
  pnpm add @revturbine/sdk
  ```
  </TabItem>
  <TabItem label="yarn">
  ```bash
  yarn add @revturbine/sdk
  ```
  </TabItem>
</Tabs>

### Peer Dependencies

The web SDK requires:

| Package | Version |
|---|---|
| `react` | `>=18.0.0` |
| `react-dom` | `>=18.0.0` |

### TypeScript

The SDK ships with full TypeScript declarations. No additional `@types/` packages are needed.

Recommended `tsconfig.json` settings:

```json
{
  "compilerOptions": {
    "strict": true,
    "moduleResolution": "bundler",
    "esModuleInterop": true,
    "jsx": "react-jsx"
  }
}
```

## Headless / Non-React

The same `@revturbine/sdk` package includes a headless entry point for use without React:

```ts
import { initRevTurbine } from '@revturbine/sdk/headless';
```

This import tree-shakes React dependencies. See the [Headless API guide](/guides/headless-api/) for usage patterns with vanilla JS, Vue, Svelte, and Angular.

## Server-Side

Server SDKs are available for backend entitlement checks and placement resolution:

| Language | Package |
|---|---|
| Node.js | `@revturbine/sdk` (headless entry) |
| Python | `revturbine` (PyPI) |

:::note
Both server SDKs are published — `@revturbine/sdk` (npm) and `revturbine` (PyPI) — and produce byte-identical decisions for the same config and user context.
:::

## CLI

The **`@revturbine/cli`** package manages your RevTurbine configuration (plans, entitlements, placements) and change sets from the terminal — verify, diff, upload, and deploy an `ExportedConfig`.

```bash
npm install -g @revturbine/cli
revturbine --help
```

Or run it without installing:

```bash
npx @revturbine/cli verify ./exported_config.json
```

See the [CLI reference](https://www.npmjs.com/package/@revturbine/cli) for the full command surface (`verify`, `diff`, `upload`, `deploy`, `export`, `status`, and the change-set lifecycle).

## Environment Variables

Set these in your environment (`.env`, CI secrets, etc.) — never commit them to source:

| Variable | Description | Required |
|---|---|---|
| `REVTURBINE_TENANT_ID` | Your tenant identifier | Yes |
| `REVTURBINE_API_KEY` | API key (prefixed `rt_live_` or `rt_test_`) | Yes |
| `REVTURBINE_ENDPOINT` | API endpoint URL | Yes (defaults to `https://api.revturbine.io`) |

```bash title=".env"
REVTURBINE_TENANT_ID=tenant_abc
REVTURBINE_API_KEY=rt_live_your_key_here
REVTURBINE_ENDPOINT=https://api.revturbine.io
```
